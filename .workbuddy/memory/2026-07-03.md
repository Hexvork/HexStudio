## RelativePathsTest CI 失败排查（只读诊断，未改文件）

**现象**：macOS CI 上 `RelativePathsTest::PathUtilComparisonTests()` 失败，`fi.exists()` 返回 FALSE，Loc 指向 `tests/src/core/RelativePathsTest.cpp(40)`。

**根因（行号铁证）**：
- CI 跑的是 HEAD(d0459db) 版本，第40行正是 `QVERIFY(fi.exists())`，该版本**无 initTestCase()**，未调用 `Engine::init(true)`。
- 本地工作区 `tests/src/core/RelativePathsTest.cpp` 有未提交修改（`git status` 显示 ` M`），给测试加了 `initTestCase(){Engine::init(true);}` + `cleanupTestCase(){Engine::destroy();}`，共 +13 行，导致 fi.exists() 偏移到第53行。
- 即：**本地修复已写好但未 commit/push，CI 仍用旧版翻车**。

**机制链**：
- `factorySamplesDir()` 返回 `m_dataDir + "samples/"` = `"data:/samples/"`。
- `data:` search path 由 `ConfigManager` 构造函数内 `initDevelopmentWorkingDir()` 注册（读 build/CMakeCache.txt 的 LMMS_SOURCE_DIR → addSearchPath("data", srcDir+"/data/")）。
- 旧版测试无 Engine::init，data: search path 注册链不完整 → `data:/samples/drums/bassdrum01.ogg` 解析失败。
- 样本文件本身存在（data/samples/drums/bassdrum01.ogg, 9922B, 在 git 中）。

**修复建议**：
1. 立即：commit + push 本地已写的 initTestCase/cleanupTestCase 修复让 CI 转绿。
2. 隐患：`Engine::init(true)` 会拉起 AudioEngine/Song/Mixer/startProcessing()，对纯路径测试过重（杀鸡用牛刀）；后续可考虑更轻量方案（手动 QDir::addSearchPath 或设 LMMS_DATA_DIR 环境变量）。`true` = renderOnly，测试场景下正确。

## 只读代码审查（找茬，未改任何源文件）

应要求对 src 核心代码做只读 bug 排查，覆盖 AudioEngine/RingBuffer/AudioEngineWorkerThread/SampleBuffer/ProjectRenderer/DataFile/ConfigManager/LmmsStyle/MidiPort。

**真实/严重 bug**
- `DataFile::upgrade_0_2_1_20070508`（L847）：`QDomElement timeEl = list.item(j)` 用错 list，应为 `vol_list.item(j)`；升级旧项目音量自动化节点会取到 instrumenttrack 列表元素。
- `LmmsStyle::darken`（L81）：函数名 darken，实现却是 `color.lighter(133)`，名实完全相反。
- `AudioEngineWorkerThread::JobQueue::run`（L86）：`++m_itemsDone` 在多 worker 线程并行执行，非原子 → 数据竞争 UB。
- `RingBuffer::write/writeAdding/writeAddingMultiplied`：未校验 `length` 是否 > m_size，调用方传大 length 时 `second=length-first` 越界 memcpy → 缓冲区溢出。
- `AudioEngine::requestChangeInModel`（L569）：非渲染线程递归调用会死锁（QMutex 非递归，同线程二次 lock 死锁）。
- `AudioEngine` 构造（L84）：`m_numWorkers = idealThreadCount()-1`，idealThreadCount 返回 -1/0/1 时 m_numWorkers 为负或 0，worker 创建逻辑异常。
- `DataFile::writeFile`（L438）：`QFile::rename` 返回值未检查即 return true，rename 失败时用户误以为保存成功却丢文件。

**中等问题**
- `AudioEngine::tryMidiClients` MidiWinMM 分支（L927-933）：注释掉 isRunning 检查后直接 return，`delete mwmm` 成死代码；注释里残留 `moss`（OSS 复制粘贴没改）。
- `ConfigManager::addRecentlyOpenedProject`（L327）：`size()>50` 后 removeLast 再 push_front，实际容量 51，应 `>=50`。
- `DataFile::copyResources`（L500）：`lastIndexOf('.')` 无扩展名时返回 -1，`truncate(-1)` 清空整个文件名。
- `LmmsStyle` 构造（L140）：`file.open()` 返回值未检查，style.css 加载失败静默无样式。
- `RingBuffer::updateSamplerate`（L311）：先 `delete[] m_buffer` 再 `new`，new 抛 bad_alloc 时 m_buffer 悬空。
- `MidiPort::invalidateCilen`（L434）：拼写错误，应为 `invalidateClient`。

**轻微/code smell**
- `LmmsStyle::drawPrimitive`（L242-244）：alpha 变量名 a75/a50/a25 与注释百分比不对应。
- `DataFile::hasLocalPlugins`（L572）：内层循环变量 i 遮蔽外层 i（-Wshadow）。
